API Reference

Last updated: Mar 26th, 2018

Foxify

Creates a Foxify instance 

const Foxify = require('foxify')

const app = new Foxify()

Foxify.constants

Foxify constants.

example:
Foxify.constants.http.NOT_FOUND // 404

Foxify.DB

go to Database API References.

Foxify.route

Creates a new Route instance.

example 1:
const route = Foxify.route()
example 2 (with prefix):
const route = Foxify.route('/prefix')

Foxify.static

This is a built-in middleware function in Foxify. It serves static files and is based on serve-static.

NOTE:

For best results, use a reverse proxy cache to improve performance of serving static assets.

example:
let options = {
  dotfiles: 'ignore',
  etag: false,
  extensions: ['htm', 'html'],
  index: false,
  maxAge: '1d',
  redirect: false,
  setHeaders: function (res, path, stat) {
    res.set('x-timestamp', Date.now())
  }
}

app.use(Foxify.static(path.join(__dirname, 'public'), options))

Foxify.dotenv

This is a built-in function in Foxify. It loads environment variables from a `.env` file into `process.env` using dotenv package.

example:
const path = require("path")

Foxify.dotenv(path.join(__dirname, '.env'))

Options

Options are some true/false variables that determine the instance behavior

example:
app.disable('x-powered-by')
app.enabled('x-powered-by') // false

app.enable('json.escape')
app.disabled('json.escape') // false
available options:
Option Description Default
x-powered-by When enabled every response will be sent with 'x-powered-by' header enabled
routing.strict When disabled the trailing slash is optional disabled
routing.sensitive When enabled the route will be case sensitive enabled
json.escape When enabled the characters <, >, and & will be escaped as Unicode escape sequences in JSON disabled

Settings

Settings are the other variables that determine the instance behavior

example:
app.set('workers', 2)
app.get('workers') // 2
available settings:
Setting Description Type Default
env Sets the process.env.NODE_ENV value String process.env.NODE_ENV or 'production'
url Sets the application url value String process.env.APP_URL or 'localhost'
port Sets the application port value Number +process.env.APP_PORT or 3000
workers Sets the number of application workers Number +process.env.WORKERS or require('os').cpus().length
json.replacer Sets the The 'replacer' argument used by JSON.stringify Function null
json.spaces Sets the The 'spaces' argument used by JSON.stringify Number null
query.parser Sets the custom query string parsing function that will receive the complete query string, and must return an object of query keys and their values Function null

View Engine

Registers the given template engine callback as View Engine

example:
app.engine('ejs', path.join(__dirname, 'views'), require('ejs').__express)

Use

Refer to route use

example:
app.use(require('morgan')('dev')) // applied for all routes

app.use('/sample_path', require('morgan')('dev')) // applied only for '/sample_path' sub routes

app.use(route) // apply routes by giving the route instance

Methods

Refer to route methods

example:
app.get('/greet', (req, res) => res.send('Hello World!'))

Start

Starts listening to the given url and port by settings

example:
app.start(() => console.log(`Foxify server running at http://${app.get("url")}:${app.get("port")} (worker: ${process.pid})`))

Route

Creates a Route instance 

const Route = require('foxify/framework/routing/route')

const route = new Route(/* prefix */) // or Foxify.route(/* prefix */)

Methods

listen to the specified method and path and send a proper respond.

api:
route[method: string](path: string, (req: http.IncommingMassage, res: http.ServerResponse, next: () => void, ...args: any[]) => any): route
example:
route.get("/path/:arg1/:arg2", (req, res, next, arg1, arg2) => res.send("Hello World!"))

OneOf

listen to the specified methods and path and send a proper respond.

api:
route.oneOf(methods: string[], path: string, ...controllers: Route.Controller[]): route
example:
route.oneOf(["GET", "POST"], "/greet", (req, res, next) => next(), (req, res) => res.send("Hello!"))

Use

Mounts the specified middleware function or functions at the specified path

api:
route.use(path: string | Route | Route.Controller, ...controllers: Route.Controller[]): route
example:
route.use(require('morgan')('dev')) // applied for all routes

route.use('/sample_path', require('morgan')('dev')) // applied only for '/sample_path' sub routes

route.use(route) // apply routes by giving the route instance
